Use this endpoint to list monitors and filter them by type, status, or label.

## Query parameters

- `limit`: Number of results to return. Default `25`.
- `cursor`: Pagination cursor from a previous response.
- `entityType`: Filter by `wallet`, `contract`, or `url`.
- `status`: Filter by `active`, `paused`, or `failing`.
- `label`: Filter by a single label.

## Example request

```bash
curl --request GET \
  --url 'https://api.chainguardai.dev/v1/monitors?limit=10&entityType=wallet&status=active' \
  --header 'Authorization: Bearer cg_live_xxxxxxxxxxxxxxxx'
```

## Error-handled example

```typescript
try {
  const response = await fetch(
    'https://api.chainguardai.dev/v1/monitors?limit=10&entityType=wallet&status=active',
    {
      headers: { Authorization: `Bearer ${process.env.CHAINGUARD_API_KEY}` }
    }
  );

  if (!response.ok) {
    const errorBody = await response.json();
    throw new Error(errorBody?.error?.message ?? 'Failed to list monitors');
  }

  const payload = await response.json();
  console.log('Monitor count:', payload.meta.count);
} catch (error) {
  console.error('List monitors failed:', error);
}
```
