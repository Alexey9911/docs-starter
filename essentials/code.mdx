## Code blocks

Use [fenced code blocks](https://www.markdownguide.org/extended-syntax/#fenced-code-blocks) by enclosing code in three backticks and follow the leading ticks with the programming language of your snippet to get syntax highlighting. Optionally, you can also write the name of your code after the programming language.

```java HelloWorld.java
class HelloWorld {
    public static void main(String[] args) {
        System.out.println("Hello, World!");
    }
}
```

## Language identifiers

Choose a language identifier that matches the syntax you are showing. This keeps highlighting accurate and makes examples easier to scan.

- `bash` for terminal commands.
- `json` for API responses.
- `typescript` or `javascript` for SDK examples.
- `yaml` for configuration files.

## Filenames and titles

If your example maps to a real file, include the filename after the language. This helps readers orient quickly.

## Command output

When you show CLI output, separate the command from the output so you can explain what happened.

```bash
chainguard scan url https://example.com
```


## Error handling in examples

When you show API calls, include an error path so readers know what to expect when something goes wrong.

```typescript
try {
  const response = await fetch('https://api.chainguard.dev/v1/scan/url', {
    method: 'POST',
    headers: {
      Authorization: `Bearer ${process.env.CHAINGUARD_API_KEY}`,
      'Content-Type': 'application/json'
    },
    body: JSON.stringify({ url: 'https://example.com' })
  });

  if (!response.ok) {
    const errorBody = await response.json();
    throw new Error(errorBody?.error?.message ?? 'Scan failed');
  }

  const payload = await response.json();
  console.log('Risk score:', payload.data.riskScore);
} catch (error) {
  console.error('Request failed:', error);
}
```
